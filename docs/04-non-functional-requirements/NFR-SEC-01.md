# NFR-SEC-01: Безопасное хранение паролей

**Идентификатор:** NFR-SEC-01

**Формулировка:**
Система должна хранить пароли пользователей в хешированном виде с использованием алгоритма bcrypt со стоимостью фактора работы (cost factor) = 12. Пароли не должны храниться или передаваться в открытом виде.

**Атрибуты:**
- Источник: lab9, Раздел 1.6 (Бизнес-риски)
- Приоритет: Critical
- Стабильность: High

**Связанные требования:**
- [US-01](../02-user-requirements/US-01.md) — Регистрация на платформе
- [FR-AU-01c](../03-functional-requirements/FR-AU-01c.md) — Проверка пароля
