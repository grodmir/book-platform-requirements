# NFR-SEC-02: Защита от перебора учётных данных

**Идентификатор:** NFR-SEC-02

**Формулировка:**
Система должна блокировать возможность входа на 15 минут после 5 последовательных неудачных попыток входа с одного IP-адреса для предотвращения атак перебором.

**Атрибуты:**
- Источник: lab9, Раздел 1.6 (Бизнес-риски)
- Приоритет: High
- Стабильность: High

**Связанные требования:**
- [US-02](../02-user-requirements/US-02.md) — Авторизация в системе
- [FR-AU-02](../03-functional-requirements/FR-AU-02.md) — Проверка учётных данных при входе
