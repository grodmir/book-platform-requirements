# NFR-SEC-03: Безопасная передача платёжных данных

**Идентификатор:** NFR-SEC-03

**Формулировка:**
Система должна передавать все платёжные данные исключительно по протоколу HTTPS с использованием TLS версии не ниже 1.2. Данные банковских карт не должны храниться на серверах платформы (за исключением токенизированных данных для функции «покупка в один клик», хранящихся в зашифрованном виде в соответствии с FR-PAY-04a).

**Атрибуты:**
- Источник: lab6, UC-08; lab9, Раздел 1.7 (Зависимости)
- Приоритет: Critical
- Стабильность: High

**Связанные требования:**
- [US-10a](../02-user-requirements/US-10a.md) — Инициирование покупки книги
- [FR-PAY-01a](../03-functional-requirements/FR-PAY-01a.md) — Перенаправление на платёжный шлюз
- [FR-PAY-04a](../03-functional-requirements/FR-PAY-04a.md) — Сохранение токена карты
